## Architecture

The library models "noisy values" — observations paired with a symbolic posterior over their true value — using SymPy for symbolic math and NumPy/SciPy for sampling.

### Core abstractions (`src/core.py`)

**`Node`** is the fundamental unit. Every noisy value is backed by a DAG of `Node` objects, each holding:
- `role`: `"latent"` (unknown true parameter), `"noise"` (random variable with a known distribution), or `"derived"` (algebraic combination)
- `symbol`: a unique SymPy `Symbol` auto-generated by `util.fresh_name()`
- `source`: a `NoiseSource` instance (for noise nodes); can reference other node symbols as parameters for hierarchical/dependent noise
- `constraints`: equations that pin latent symbols (e.g., `obs = theta + eps`)
- `depends_on`: parent nodes, forming the DAG

**`NoisyValue`** wraps a `(obs, root_node)` pair. Subclasses enforce the observed type:
- `NoisyFloat` — real-valued observations; supports arithmetic, exp/log/sqrt, rounding
- `NoisyInt` — integer observations; has `resample(source)` to attach a dependent `NoiseSource`
- `NoisyBool` — boolean observations; supports `&`, `|`, `~`
- `NoisyNumber` — shared numeric base (comparison operators that return `NoisyBool`, `guarded()`)

Arithmetic operators on `NoisyValue` subclasses call `bin_op` / `unary_op`, which compose both the observed scalar and the symbolic expression in tandem.

**`NoisyValueSampler`** (returned by `noisy_value_sampler()`) does the heavy lifting:
1. Collects all latent symbols and constraints across a joint set of values
2. Solves latent symbols symbolically via `sympy.solve` (static substitutions, computed once)
3. Separates noise nodes into *independent* (no `depends_on`) and *dependent* (hierarchical)
4. On each `.sample(n)` call, bulk-samples all independent noise via NumPy, then resolves dependent nodes in topological order using `source.instantiate(resolved).sample(rng)`, then evaluates expressions via `sp.lambdify`
5. Returns a `SampleBatch` wrapping a 1-D NumPy array

### Supporting modules

- **`src/noise.py`** — `NoiseSource` base class and two concrete subclasses: `NormalNoiseSource`, `BinomialNoiseSource`. Each implements `free_symbols` (SymPy symbols its parameters depend on), `instantiate(resolved)` (substitutes resolved values into parameters), `sample(rng, size)` (NumPy sampling), and `sympy_rv()` (returns a SymPy RV for visualization). Use the public constructors `gaussian(loc, scale)` and `binomial(n, p)` — parameters can be plain numbers, SymPy expressions, or `NoisyFloat`/`NoisyInt` values.
- **`src/analysis.py`** — `NoisyContingencyTable` wraps a 2-D array of `NoisyNumber`s and provides `chi_squared()`, `odds_ratio()`, and `with_sampling_uncertainty()` (the last one adds binomial sampling uncertainty row-by-row, which is the key differential-privacy use case). Also exposes `noisy_min` / `noisy_max`.
- **`src/array.py`** — `float_array_sampler` / `sample_float_array` for sampling tensor-shaped collections of noisy values jointly.
- **`src/visual.py`** — `plot_posterior()` renders posterior densities deterministically via tensor-product Gauss-Legendre quadrature in quantile space (no Monte Carlo), then weighted KDE. Uses `source.sympy_rv()` to get quantile functions for each noise source. Requires `matplotlib` (dev dependency).
- **`src/util.py`** — `fresh_name()` / `reset_name_provider()` for globally unique SymPy symbol names. Tests call `reset_name_provider()` via `conftest.py` to avoid symbol collisions.

### Key design constraint

Every `NoisyValue` carries its full probabilistic provenance in the Node DAG. Sampling is always *joint*: shared nodes (same `symbol`) are sampled once per draw and reused, so correlated values stay correlated. This is essential for the differential-privacy contingency table use case where row marginals must be consistent.
